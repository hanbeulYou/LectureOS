# Document Review Checklist

- Status: Active
- Version: 0.1
- Last Updated: 2026-07-13

## 1. 제품 의도

- [ ] 실제 사용자의 문제를 정확히 설명한다.
- [ ] 사용자가 말하지 않은 목표를 임의로 추가하지 않았다.
- [ ] 단순 자막 생성기로 축소되지 않았다.
- [ ] 지나치게 범용 플랫폼으로 과장되지 않았다.
- [ ] 현재와 목표 워크플로가 구분된다.
- [ ] Final Cut을 유지하는 이유가 정확하다.

## 2. 결정 상태

- [ ] 확정 결정과 가설이 구분된다.
- [ ] Open Question을 확정 사항처럼 쓰지 않았다.
- [ ] 이전 승인 결정과 충돌하지 않는다.
- [ ] 결정 변경 이유가 기록되어 있다.
- [ ] 필요한 ADR 대상이 표시되어 있다.

## 3. 범위

- [ ] 장기 제품 범위와 v1 범위가 구분된다.
- [ ] v1이 후속 확장을 막지 않는다.
- [ ] v1이 전체 제품을 한 번에 구현하려 하지 않는다.
- [ ] Non-goals가 명확하다.
- [ ] 자동 컷편집과 편집 후보 제안이 구분된다.
- [ ] Final Cut 연동 단계가 명확하다.

## 4. 데이터와 시간축

- [ ] 원본 미디어가 물리적 사실의 원천이다.
- [ ] 원본 시간축의 단어/발화 데이터가 보존된다.
- [ ] raw 인식문, 교정문, 자막문이 구분된다.
- [ ] SRT를 source of truth로 취급하지 않는다.
- [ ] 편집 후 시간축이 원본 시간축과 구분된다.
- [ ] 파생 결과를 다시 생성할 수 있다.

## 5. AI 역할

- [ ] LLM이 타임스탬프를 직접 결정하지 않는다.
- [ ] LLM 출력에 검증 단계가 있다.
- [ ] 음성에 없는 내용을 추측해 확정하지 않는다.
- [ ] 불확실한 변경은 review 대상으로 남긴다.
- [ ] ASR·LLM·alignment 모델 교체 가능성을 유지한다.
- [ ] 특정 공급자를 제품 원칙으로 고정하지 않는다.

## 6. 기술적 현실성

- [ ] 수십 GB 원본을 불필요하게 복제하지 않는다.
- [ ] 2시간 이상 영상 처리 실패를 고려한다.
- [ ] 캐시와 재개 가능성이 고려된다.
- [ ] 현재 하드웨어 환경을 반영한다.
- [ ] 기존 코드의 한계를 이상적 기능으로 오인하지 않는다.
- [ ] 검증되지 않은 API 기능을 사실처럼 단정하지 않는다.

## 7. 문서 품질

- [ ] 제목과 목적이 분명하다.
- [ ] Status, Version, Last Updated가 있다.
- [ ] 용어가 다른 문서와 일치한다.
- [ ] 같은 내용을 과도하게 반복하지 않는다.
- [ ] 관련 문서 링크가 있다.
- [ ] Open Questions가 있다.
- [ ] Change Log가 있다.
- [ ] 구현 세부사항이 제품 문서에 과도하게 섞이지 않았다.

## 8. 커밋 전

- [ ] 해당 문서만 stage했다.
- [ ] `git diff --cached`를 확인했다.
- [ ] 커밋 메시지가 설계 의도를 설명한다.
- [ ] 미승인 문서나 코드가 함께 stage되지 않았다.
- [ ] Codex가 임의로 수정한 핵심 결정이 없는지 확인했다.

## 9. 승인 상태

- Draft
- In Review
- Approved
- Superseded

Approved 문서를 의미 있게 변경할 때는 Version과 Change Log를 갱신한다.

- 체크리스트를 PR 템플릿에 포함할지
