## Purpose

이 문서는 LectureOS가 누구를 위해 어떤 문제를 해결하며, 현재 무엇을 제품의 안과 밖으로 보는지 정의하는 제품 계약이다.

## Product Definition

LectureOS는 한국어 장시간 교육 강의를 촬영한 뒤 발생하는 반복적인 후반작업을 줄이는 제품이다.

원본 미디어와 원본 시간축 기반 발화·단어 데이터를 바탕으로 Text Pipeline과 Edit Pipeline을 동등한 핵심 흐름으로 제공한다. 사람이 Transcript와 학생용 Subtitle을 교정·검수하는 동시에, 강의 구간 분석과 컷편집용 라벨링에서 생성된 Edit Candidate를 검수하고 편집 결정을 남길 수 있도록 돕는다.

LectureOS는 사람의 교육적·편집적 판단을 대체하는 범용 자동 제작 플랫폼이 아니다.

## Core Problem

초기 사용자는 긴 강의 영상을 반복해서 재생하며 다음 문제를 동시에 해결한다.

- 음성 인식 오류와 전문용어를 찾고 고친다.
- 학생이 읽기 어려운 자막을 다시 나누고 타이밍을 조정한다.
- 불필요하거나 확인이 필요한 구간을 찾는다.
- 원본과 편집 후 시간축 사이에서 자막을 맞춘다.
- 화면 편집과 내용 검수를 오가며 같은 구간을 여러 번 확인한다.

개별 자동화 도구가 있어도 이 정보와 결정이 분리되어 있어 총 후반작업 시간이 충분히 줄지 않는 것이 핵심 문제다.

## Primary User and Initial Context

Primary User는 장시간 강의 영상을 직접 후반작업하는 한국어 교육자 또는 소규모 교육 콘텐츠 운영자다.

초기 사용 맥락은 다음과 같다.

- 한 명의 주강사가 진행하는 한국어 교육 강의
- 한 편이 대체로 1시간 이상인 긴 형식의 녹화물
- 전문 편집 인력이 없거나 제한된 반복 제작 환경
- 자막 검수와 컷 판단을 같은 재생 과정에서 수행하는 흐름
- 고유명사와 전문용어의 인식 오류가 중요한 품질 문제인 환경

현재 사용자가 특정 데스크톱 편집 도구와 로컬 Mac 환경을 사용한다는 사실은 출발점의 제약이지 LectureOS의 영구 정체성은 아니다.

## Product Goals

1. 강의 한 편을 완성하는 총 검수 시간을 줄인다.
2. 중요한 의미 오류와 시간축 오류를 사람이 더 빨리 발견하게 한다.
3. 원본과 사용자 결정을 보존해 파생 결과를 다시 생성할 수 있게 한다.
4. 자막의 정확성과 가독성을 높여 학생의 학습 경험을 개선한다.
5. 컷편집용 라벨링과 Edit Candidate를 통해 편집 후보 검토 시간을 줄인다.
6. 현재 핵심 사용 사례에서 반복 가능한 가치를 먼저 검증한다.

## In Scope

- 원본 미디어를 기준으로 한 발화·단어 위치의 추적
- raw 인식문, 사람이 교정한 Transcript, 학생용 Subtitle의 구분
- Transcript와 Subtitle의 생성·교정·검수 지원
- 강의 구간 분석과 컷편집용 라벨링
- 원본 시간축과 연결된 Edit Candidate 생성
- 오류, 불확실성, 자막 변경, 편집 후보를 Review Item으로 제시
- 자막과 Edit Candidate를 함께 다루는 통합 Review
- 원본 및 편집 후 시간을 연결하는 Timeline 정보
- 사용자의 수정과 편집 결정 보존
- 승인된 자막과 편집 결정을 Final Cut 이전에 준비
- 파생 결과를 다시 생성하고 비교할 수 있는 작업 흐름
- 장시간 한국어 교육 강의에 대한 검수 시간 감소

## Out of Scope

- 촬영, 화면 녹화, 방송 송출 도구
- 전문 비선형 영상 편집기의 시각 편집 기능 전체
- LMS, 수강생 관리, 결제, 출결, 강좌 판매
- 스트리밍 또는 콘텐츠 배포 플랫폼
- 범용 OCR 플랫폼
- 자체 ASR·LLM 기반 모델 개발
- 사람의 확인 없는 교육적 판단과 자동 삭제
- 모든 언어, 모든 영상 장르, 모든 편집 환경의 동시 지원

## Product Boundaries

- LectureOS는 단순 자막 생성기에 머물지 않지만, 모든 후반작업을 소유하지도 않는다.
- 외부 인식 모델과 편집 도구는 교체하거나 연동할 수 있는 구성요소다.
- 현재 범위는 자막 품질 개선과 컷편집용 라벨링을 동등한 핵심으로 두고, 분석, 텍스트, 시간축, 검수 정보와 사용자 결정의 연결에 집중한다.
- LectureOS는 Final Cut 이전에 자막과 Edit Candidate의 검수를 지원하고 승인된 결과를 준비한다.
- Final Cut은 현재 핵심 사용 환경에서 화면 편집, 최종 컷 확인, 렌더링을 담당한다. 특정 편집 도구는 LectureOS의 영구 정체성이 아니다.
- 구체적인 단계별 기능과 출시 순서는 향후 Product Requirements와 Roadmap에서 결정한다.
- 데이터 구조의 정확한 이름과 필드는 향후 Glossary와 Data Model에서 결정한다.

## Success

LectureOS는 다음 변화가 실제 작업에서 확인될 때 성공한다.

- 기존 흐름보다 전체 검수와 수정 시간이 줄어든다.
- 중요한 의미 오류와 불확실한 구간을 더 쉽게 찾는다.
- 사용자가 원본부터 같은 작업을 다시 하지 않아도 된다.
- 사용자 수정이 남고, 조건이 바뀌면 파생 결과를 다시 만들 수 있다.
- 학생이 읽는 자막의 정확성과 가독성이 유지되거나 향상된다.

## Failure

다음 중 하나가 지속되면 제품은 실패한 것이다.

- 자동화 결과를 확인하느라 기존보다 시간이 더 든다.
- 자연스럽지만 근거 없는 변경이 중요한 오류를 만든다.
- 원본, 시간축, 교정, 자막의 관계를 추적할 수 없다.
- 사용자 수정이나 편집 결정이 유실된다.
- 특정 모델이나 도구 때문에 핵심 작업 흐름을 바꿀 수 없다.
- 범위를 넓히느라 현재 핵심 사용 사례에서 실질적 가치를 만들지 못한다.

## Future Optionality

LectureOS는 향후 다른 언어, 팀 협업, 오픈소스 배포 또는 SaaS 운영으로 발전할 가능성을 막지 않는다. 그러나 어느 방향도 현재 확정된 제품 범위나 사업 모델이 아니다.

확장 가능성은 현재 핵심 사용 사례의 품질과 검수 시간 감소를 희생할 근거가 될 수 없다.

- 현재 핵심 사용 사례를 넘어서는 첫 확장 축은 사용자, 언어, 협업 중 무엇인가?

- 특정 편집 도구와의 연동은 어느 시점에 제품 범위로 승인할 것인가?
